# 日志选择复制功能设计

## 概述

在日志页面增加条目选择功能，支持部分复制。统一名称为"复制"而非"拷贝"。

## 需求

1. 增加日志条目的选择功能，实现部分复制
2. 统一名称为"复制"，而不是"拷贝"
3. 支持复选框多选
4. 支持 Shift+点击连续范围选择
5. 复制格式保持与现有"拷贝全部"相同的完整格式

## 架构

### 状态管理

```ts
let selectMode = false;       // 是否在选择模式
let selectedIds = new Set();  // 选中的日志索引
let lastClickedIndex = -1;    // 上次点击的索引（用于Shift范围选择）
```

### UI 状态

- **默认状态**：顶部显示「复制」按钮
- **选择模式**：点击「选择」进入，每条日志前出现复选框
- **选择状态**：顶部显示「复制选中 (N)」和「取消」按钮

### 交互流程

1. 用户点击「选择」按钮进入选择模式
2. 每条日志前显示复选框
3. 用户点击复选框选择/取消单条日志
4. 用户 Shift+点击进行范围选择
5. 顶部显示选中数量
6. 用户点击「复制选中 (N)」复制选中的日志
7. 复制完成后自动退出选择模式

### 关键实现

- 复选框在选择模式下显示，默认隐藏
- Shift+点击时计算范围：`Math.min(last, current)` 到 `Math.max(last, current)`
- 复制时将选中条目按原格式拼接
- 「复制」按钮在未选中时复制全部
