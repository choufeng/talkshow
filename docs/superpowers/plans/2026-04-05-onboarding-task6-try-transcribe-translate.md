# 子任务 6：步骤 5 试用转写 & 步骤 6 试用翻译

**所属设计**: `2026-04-05-onboarding-wizard-design.md`  
**依赖**: 子任务 2（向导框架）、子任务 4（模型下载完成）  
**可并行**: 否（需要模型和 Provider 都就绪）

---

## 目标

实现端到端验证的两个试用步骤：转写和翻译。用户实际操作功能，确认端到端可用。

---

## 任务清单

### T6.1: TryTranscriptionStep（步骤 5）

**文件**: `src/lib/components/onboarding/steps/TryTranscriptionStep.svelte`

**操作**:
- 显示提示文字：「按 `Ctrl+\` 录一段话，然后再次按下停止」
- 显示快捷键按键样式提示
- 录音状态指示（监听 Tauri 录音事件）
- 录音停止后，等待转写结果（通过 Tauri 事件或轮询）
- 转写结果展示区域
- 收到转写结果 → 通知步骤完成，启用「下一步」
- 30 秒超时未检测到操作 → 显示「未检测到操作」提示 + 「重试」按钮 + 「跳过」选项
- 跳过不影响引导完成

### T6.2: TryTranslationStep（步骤 6）

**文件**: `src/lib/components/onboarding/steps/TryTranslationStep.svelte`

**操作**:
- 显示提示文字：「选中上一步的转写文本，按 `Ctrl+Shift+T`」
- 显示快捷键按键样式提示
- 显示待翻译的文本（步骤 5 的转写结果，如无则显示示例文本）
- 等待翻译结果返回
- 翻译结果展示区域（原文 → 译文）
- 收到翻译结果 → 通知步骤完成，启用「下一步」
- 30 秒超时未检测到操作 → 显示「未检测到操作」提示 + 「重试」按钮 + 「跳过」选项
- 跳过不影响引导完成

### T6.3: CompletionStep（步骤 7）

**文件**: `src/lib/components/onboarding/steps/CompletionStep.svelte`

**操作**:
- 显示成功图标
- 文字：「配置完成！TalkShow 已准备就绪。」
- 按钮「开始使用」→ 调用 `invoke('set_onboarding_completed', { completed: true })` → 触发页面切换到主界面

---

## 验证

```bash
npm run check
npm run dev
```

手动验证：
1. 完成前置步骤后进入试用转写
2. 按快捷键录音，确认录音状态指示显示
3. 停止录音，确认转写结果展示
4. 进入试用翻译，按快捷键触发翻译
5. 确认翻译结果展示
6. 确认完成页正常，点击「开始使用」进入主界面

---

## 验收标准

- 试用转写：录音 → 转写结果展示 → 步骤完成
- 试用翻译：触发翻译 → 翻译结果展示 → 步骤完成
- 30 秒超时有重试和跳过选项
- 跳过不影响引导完成
- 完成页点击后正确进入主界面
